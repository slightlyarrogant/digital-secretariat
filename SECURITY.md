# Security policy

Do not open a public issue containing credentials, customer data, email content, private hostnames,
or exploit details. Report a suspected vulnerability privately to the repository owner through the
GitHub account's established security contact.

For the product threat model and deployment controls, see [`docs/SECURITY.md`](docs/SECURITY.md).
Security fixes must preserve audit history and include a regression test. Never demonstrate a mail
or prompt-injection issue against a real customer address.
